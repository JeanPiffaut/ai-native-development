## Contexto de equipo

Proyecto individual — un solo developer que actúa como maintainer del framework y como usuario en los proyectos que lo adoptan. Las decisiones no requieren consenso de equipo; el agente propone y el developer aprueba.

## Externos relevantes

No hay stakeholders externos activos en esta etapa. Si el framework se abre a contribuciones externas, este archivo se actualiza para reflejar el proceso de revisión de PRs.
